Orquestando Agentes de IA Autonomos con Roles

Que es un Agente de IA?

Una entidad autonoma que usa Modelos de Lenguaje (LLMs) para razonar, usar herramientas y completar tareas complejas para lograr un objetivo especifico.

Tipos de Agentes de IA

Segun IBM, los agentes van desde sistemas reflejos simples hasta sistemas de aprendizaje complejos.

  • Agentes Reflejos Simples
  • Agentes Reflejos Basados en Modelo
  • Agentes Basados en Objetivos
  • Agentes Basados en Utilidad
  • Agentes de Aprendizaje

crewAI se enfoca principalmente en la colaboracion Basada en Objetivos y Basada en Utilidad.

Fuente: IBM Think Topics

Agente Individual vs. Multi-Agente

Un Solo LLM

Pedirle a una sola persona que haga todo (Investigar, Analizar, Escribir, Revisar).

Multi-Agente (crewAI)

Asignar tareas a agentes especializados que colaboran como un equipo profesional.

Analogia: Una Redaccion de Noticias profesional. Tienes un Reportero, un Editor y un Verificador de datos trabajando juntos, en lugar de un solo blogger intentando hacerlo todo.

Como Funciona crewAI

Construido sobre LangChain con un principio de diseno modular.

Componentes Principales:

  • Agents: Los trabajadores especializados.
  • Tasks: Las asignaciones.
  • Tools: El equipamiento (Busqueda web, APIs).
  • Process: El flujo de trabajo (Secuencial/Jerarquico).

Basicamente, defines el "Quien", "Que" y "Como", y luego el Crew gestiona la ejecucion.

La Colaboracion y el Proceso

El Agent

"Quien hace el trabajo?" — Piensa en ello como una Descripcion de Puesto.

researcher:
  role: "Investigador Senior de Datos de {topic}"
  goal: "Descubrir desarrollos de vanguardia en {topic}"
  backstory: "Eres un investigador experimentado con talento..."

El Task

"Que hay que hacer?" — Debe asignarse a un agente.

research_task:
  description: "Realizar una investigacion exhaustiva sobre {topic}..."
  expected_output: "Una lista con 10 puntos."
  agent: researcher

El Tool

"Que recursos pueden usar?"

  • Capacidades: Busqueda Web, Lectura/Escritura de Archivos, llamadas a APIs.
  • Accion: Sin herramientas, los agentes solo pueden "pensar" (razonar). Con herramientas, pueden "actuar".
  • Personalizacion: Usa herramientas integradas o crea tus propias herramientas personalizadas en Python.

El Crew

"Como opera el equipo?"

  • Orquestacion: Coordina agentes y tareas.
  • Procesos:
    - Secuencial: Paso a paso (A → B → C)
    - Jerarquico: Un agente Manager delega el trabajo.
  • Kickoff: Inicia el proceso con entradas para obtener el resultado final.
